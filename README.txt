1. Запуск:

	Запустити bin\Debug\net8.0\puzzles.exe

	Або відкрити як проєкт puzzles.sln.

	Вхідний файл "source.txt" повинен бути поруч з "puzzles.exe"

2. Можливі налаштування:

	Якщо відома приблизна кількість пазлів, які можна скласти з набору, її можна задати для пришвидшення пошуку (Set searchable depth).

	З поточного файлу максимум — 67.

	Напрямок "справа наліво" заданий як стандартне значення, в даному випадку працюэ швидше.

3. Короткий опис:

	Для структуризації використовується зв’язний список, у кожній його ноді міститься пазл.

	Кожного разу як початковий пазл використовується наступний із списку.

	Алгоритм будує дерево за допомогою рекурсивної функції. У випадках розгалужень, коли можна підставити декілька варіантів, створюється додаткова альтернативна гілка, яка продовжує побудову.

	Для пошуку найдовшої гілки в дереві використовується також рекурсивна функція, яка повертає найдовшу послідовність.

	При знаходженні найдовшого варіанту серед дерев послідовність зберігається, поки не знайдеться довша.
